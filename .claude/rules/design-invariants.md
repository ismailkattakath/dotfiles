# Dotfiles Design Invariants

These rules are non-negotiable. Violating any of them breaks the container/Codespaces workflow.

## Templates — identity variables + the macOS prefs hash guard

`.tmpl` files are permitted for two narrow purposes only:

1. **Identity injection** — files referencing identity/config variables from chezmoi data:
   `{{ .name }}`, `{{ .email }}`, `{{ .signingKey }}`.
2. **The macOS prefs `run_onchange` guard** — `run_onchange_macos-prefs.sh.tmpl` embeds
   `{{ include "macos/…" | sha256sum }}` hashes so chezmoi re-imports the plists only when they change.

**No platform-switching `{{ if }}` / `{{ if eq .chezmoi.os "darwin" }}` conditional blocks, ever.**

Identity variables are defined in:
- Host: `~/.config/chezmoi/chezmoi.toml` `[data]` section.
- Containers: generated by `install.sh` from env vars (`GIT_AUTHOR_NAME`, `GIT_AUTHOR_EMAIL`, `GIT_SIGNING_KEY`).

Adding a new identity variable requires updating both places. Never hardcode an email/identity
value in a tracked file — pull it from chezmoi data (e.g. `dot_gitconfig.tmpl` uses `{{ .email }}`).

**Every variable any `*.tmpl` references MUST be written by `install.sh`'s non-interactive
block.** chezmoi runs with `missingkey=error`, so a single undefined variable aborts the entire
`chezmoi apply` — in Codespaces that means *no dotfiles install at all*. The container/Codespaces
path has no host `chezmoi.toml` to fall back on, so `install.sh` is the only source of these vars.

## Secrets & privacy — never commit

- No secrets, tokens, API keys, or credentials of any kind — not even as examples.
- No real email addresses hardcoded in tracked files — use chezmoi data variables or GitHub noreply.
- No absolute personal paths (`/Users/<actual-username>/…`); macOS prefs must be scrubbed of
  usernames and security-scoped path bookmarks before committing (`NSOSPLastRootDirectory`,
  `BackgroundImageBookmark` leak `/Users/<name>` inside base64 — strip them).
- The gitleaks pre-commit hook (`.githooks/pre-commit`) and the `secret-scan.yml` CI workflow
  enforce this; run `/audit` before pushing.

## Never add

- `{{ if }}` / `{{ if eq .chezmoi.os "darwin" }}` or any platform-conditional template blocks
- `.zshrc`, `.zprofile`, or any shell config — devcontainer features own the shell
- `Brewfile` — package installation is out of scope
- `run_once_*` scripts — features handle installation (note: `run_onchange_*` IS used, for macOS prefs)
- Secrets, tokens, or hardcoded real emails of any kind — not even as examples
- Real absolute personal paths in committed files (e.g. `/Users/<actual-username>/`); generic
  placeholders (`/Users/USER`) only, and only where unavoidable (macOS prefs)
- `UseKeychain yes` in `dot_ssh/config` — the container-applied SSH config must stay keychain-free
  (machine-specific host entries go in the gitignored `~/.ssh/config.local`)

## Always ensure

- `dot_gitconfig` credential helper uses `!gh auth git-credential` — never the Homebrew full path
- `dot_gitconfig` uses `gpg.format = ssh` and `user.signingkey` set to the ed25519 public key
- `install.sh` is listed in `.chezmoiignore` — it must not be applied to `~/`
- All files are plain except `*.tmpl` identity files — chezmoi copies plain files as-is
- `dot_ssh/config` begins with `Include ~/.ssh/config.local` for machine-specific hosts

## Credential helper rule

```
# CORRECT — resolves gh from PATH, works everywhere
helper = !gh auth git-credential

# WRONG — Homebrew path, breaks inside Linux containers
helper = !/opt/homebrew/bin/gh auth git-credential
```

## Scope

This repo manages two things:
1. **Container / Codespaces dotfiles** — git identity, SSH config, editor config, gh config —
   applied via chezmoi inside devcontainers/Codespaces (and on the host).
2. **Host macOS terminal preferences** — `macos/*.plist` (Terminal + iTerm2), imported on Darwin
   only via `run_onchange_macos-prefs.sh.tmpl` (`defaults import`); a no-op on Linux.

## What this repo is NOT

- Not a shell customisation repo (no prompt, no aliases, no functions, no `.zshrc`/`.zprofile`)
- Not a package manager (no Brewfile, no install scripts beyond chezmoi bootstrap)
- Not a secret store (no tokens, no keys, no credentials, no hardcoded personal emails)

## File naming

- `dot_X` → `~/.X`
- `dot_config/Y/Z` → `~/.config/Y/Z`
- `dot_ssh/config` → `~/.ssh/config`
